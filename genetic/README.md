# Пакет `genetic`

Генетический алгоритм для подбора весов линейной политики змейки: максимизация скаляра **J(θ)** из одного rollout на фиксированном сиде игры (см. `player.rollout.simulate`).

## Зависимости

Из корня проекта (`pyproject.toml`): `tqdm`; используются модули `game`, `player`.

## Быстрый старт

Из корня репозитория:

```bash
uv run python run_genetic_optimization.py --training-seed 1 --snake-game-seed 42 --rollout-workers 0
```

Программно:

```python
from pathlib import Path
from genetic import GeneticHyperparams, GeneticLearn

hp = GeneticHyperparams(training_seed=1)
learn = GeneticLearn(
    hp,
    field_size=(10, 10),
    snake_game_seed=42,
    results_dir=Path("out/my_run"),
    verbose=True,
)
best_theta, best_J, aborted = learn.run()
```

## Два сида (важно)

| Параметр | Где задаётся | Роль |
|----------|--------------|------|
| **Сид игры (rollout)** | `GeneticLearn(..., snake_game_seed=...)` | Одно и то же целое передаётся в **каждый** вызов `simulate(..., seed=snake_game_seed)` для всех поколений и всех особей: одна и та же воспроизводимость поля и RNG игры/политики на эпизод. |
| **Сид обучения (ГА)** | `GeneticHyperparams.training_seed` | Зерно для `random.Random` **только** операторов ГА: начальная популяция (см. `initial_weights` / `random_theta`), турнирный отбор, кроссовер, мутация. На rollout не влияет. |

## Гиперпараметры `GeneticHyperparams`

Все поля датакласса (кроме обязательного `training_seed`) имеют значения по умолчанию в коде; ниже — смысл и ограничения.

| Поле | Тип (по умолчанию) | Описание |
|------|-------------------|----------|
| **`training_seed`** | `int` (обязательно) | Начальное зерно RNG для операторов ГА. Меняет траекторию поиска (инициализация, случайность отбора и потомков), но не меняет сид одного rollout при фиксированном `snake_game_seed`. |
| **`population_size`** | `int`, по умолчанию `50` | Число особей (наборов весов `w_food, w_danger, w_space, w_wall`) в поколении. Должно быть **≥ 2**. |
| **`generations`** | `int`, по умолчанию `30` | Сколько поколений эволюции выполнить (внешний цикл: оценка популяции → отбор/скрещивание/мутация → следующее поколение). Должно быть **≥ 1**. |
| **`max_steps`** | `int`, по умолчанию `200` | Базовый лимит шагов змейки за rollout (аргумент `max_steps` в `simulate_rollout`). Должно быть **≥ 1**. |
| **`stretch`** | `bool`, по умолчанию `False` | **Вытягивание**: если после `max_steps` игра ещё `IN_PROGRESS`, добавляются блоки по `stretch_chunk` шагов, пока за блок съедено **≥ 1** яблоко (иначе стоп — зацикливание). |
| **`stretch_chunk`** | `int`, по умолчанию `100` | Размер одного блока дополнительных шагов при вытягивании. Должно быть **≥ 1**. |
| **`randomize_game_seed_per_generation`** | `bool`, по умолчанию `False` | Если `True`, на **каждое поколение** новый `snake_game_seed` для всех особей (детерминированно от RNG с `training_seed`). Иначе везде сид из `GeneticLearn(..., snake_game_seed=...)`. |
| **`crossover_prob`** | `float`, по умолчанию `0.7` | Вероятность при создании потомка применить **кроссовер** между двумя родителями (иначе копируется один из родителей с вероятностью 0.5). Диапазон смысла — обычно `[0, 1]`. |
| **`mutation_prob`** | `float`, по умолчанию `0.15` | Для **каждого** из четырёх весов независимо: с этой вероятностью добавляется гауссов шум (см. `mutation_sigma`), затем вес клипается в допустимый диапазон (как у `random_theta` в `player.policy`). |
| **`mutation_sigma`** | `float`, по умолчанию `0.25` | Стандартное отклонение гауссианы при мутации одного веса (если сработала `mutation_prob` для этого гена). |
| **`elite_count`** | `int`, по умолчанию `2` | Сколько лучших по J особей текущего поколения **без изменений** копируется в следующее. Должно удовлетворять **0 ≤ elite_count < population_size**. |
| **`tournament_size`** | `int`, по умолчанию `3` | Размер выборки в **турнирном отборе**: случайно выбирается `k` особей, родителем становится лучшая по J среди них. Должно быть **≥ 1**; фактически используется `min(k, population_size)`. |
| **`initial_weights`** | `tuple[float, float, float, float] \| None`, по умолчанию `None` | Стартовая **точка** в пространстве весов `(w_food, w_danger, w_space, w_wall)` для **нулевого поколения**. Если `None`, поколение 0 по-прежнему задаётся через `random_theta` (равномерные независимые диапазоны из `player.policy`). Если задано, каждая особь поколения 0 получается как центр плюс случайный гауссов шум (см. `initial_spread` / `initial_spread_per_weight`), затем **клип** в те же границы, что и у мутации / `random_theta`. |
| **`initial_spread`** | `float`, по умолчанию `0.35` | Одна общая **σ** гауссианы для всех четырёх весов при возмущении вокруг `initial_weights`. Используется только если `initial_weights` задан и **`initial_spread_per_weight` равен `None`**. Должно быть **> 0** в этом режиме. |
| **`initial_spread_per_weight`** | `tuple[float, float, float, float] \| None`, по умолчанию `None` | Если задано вместе с `initial_weights`, для каждого гена `i` шум — `N(0, initial_spread_per_weight[i])` (независимые гауссианы). Все четыре компонента должны быть **> 0**. В этом случае поле **`initial_spread` игнорируется** для инициализации (оно может остаться в `meta.json` как часть конфигурации). |
| **`rollout_workers`** | `int`, по умолчанию `0` | Параллельная оценка популяции: **`0`** — авто (число процессов не больше ядер, размера популяции и 32); **`1`** — строго по одной особи в главном процессе (удобно для досрочной остановки между особями); **`≥2`** — верхняя граница воркеров `ProcessPoolExecutor`. В многопроцессном режиме запрос остановки обрабатывается только **между** поколениями. |

Сериализация всех этих полей (вместе с `training_seed`) попадает в `meta.json` прогона в блок `genetic_hyperparams`.

## Параметры конструктора `GeneticLearn`

Первый аргумент — экземпляр `GeneticHyperparams`; остальные только именованные (`*`).

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| **`hyperparams`** | — | Объект `GeneticHyperparams` со всеми гиперпараметрами ГА (см. таблицу выше). |
| **`field_size`** | `(10, 10)` | Размер поля змейки `(height, width)` — передаётся в `simulate` для каждой оценки особи. |
| **`snake_game_seed`** | обязателен | Сид rollout (игра + согласованный RNG политики), **один на весь** прогон обучения для всех особей и поколений. |
| **`results_dir`** | обязателен | Каталог для артефактов (создаётся при необходимости). |
| **`verbose`** | `True` | Если `True`: подробные сообщения в stdout и прогресс-бар `tqdm` при оценке популяции в каждом поколении. Если `False` — тихий режим (удобно для скриптов и CI). |

## Результаты `run()`

В каталоге `results_dir` создаются:

| Файл | Содержимое |
|------|------------|
| **`meta.json`** | Версия, `snake_game_seed`, размер поля, словарь `genetic_hyperparams`; после завершения `run()` — поле **`run_aborted`** (`true`/`false`). |
| **`results.csv`** | Колонки `w_food`, `w_danger`, `w_space`, `w_wall`, `J`, **`steps`**, **`snake_game_seed`** (сид rollout для этой строки), `gen_number`, `snake_number`. При остановке во время поколения последнее поколение может быть **неполным**. |
| **`best.json`** | Лучшая по J особь на момент остановки: четыре веса, `J`, поле **`aborted`**. Если ни одного rollout не было, для `J` выполняется один расчёт `simulate` по текущему `best_ever`. |

Метод возвращает `(best_theta, best_J, aborted)` — лучшие веса, значение цели и признак досрочной остановки.

## Логика ГА (кратко)

1. Инициализация поколения 0: при `initial_weights is None` — `random_theta(rng)` для каждой особи; иначе — центр `initial_weights` плюс гауссов шум (`initial_spread` или поосевые `initial_spread_per_weight`), затем клип в допустимые интервалы весов. `rng = Random(training_seed)`.
2. Поколение: для каждой особи считается `J` и фактическое число шагов через `simulate_rollout(...)` (с опциональным вытягиванием).
3. Лучшие `elite_count` особей копируются в следующее поколение.
4. Остальные слоты заполняются потомками: два родителя — турнир; с вероятностью `crossover_prob` — арифметический кроссовер по генам, иначе клон одного родителя; затем мутация по `mutation_prob` / `mutation_sigma` и обрезка весов в тех же границах, что и у `random_theta`.

## Скрипт `run_genetic_optimization.py`

CLI-обёртка над теми же гиперпараметрами; флаги см. `--help`. Флаги **`--stretch`**, **`--stretch-chunk`**, **`--randomize-game-seed-per-generation`**. Флаг **`--rollout-workers`** задаёт параллельную оценку популяции. Флаг **`--quiet`** отключает подробный вывод и `tqdm`.

## Браузерный UI (`ui/genetic_app.py`)

Интерактивная настройка гиперпараметров, прогресс-бар и график по поколениям, воспроизведение лучшей партии (как в `ui/app.py`):

```bash
uv run streamlit run ui/genetic_app.py
```

Обучение вызывает `GeneticLearn.run(progress_callback=...)`: по ходу эпох приходят события `snake_evaluated` и `generation_done` (см. докстринг `GeneticLearn.run`). Для принудительной остановки используйте аргумент `interrupt_check` (например `threading.Event.is_set`); метод возвращает `(best_theta, best_J, aborted: bool)`, при остановке сохраняются `results.csv`, `best.json` и в `meta.json` добавляется поле `run_aborted`.

Старт вокруг заданной точки весов:

```bash
uv run python run_genetic_optimization.py --initial-weights "1.0,-2.0,0.5,0.0" --initial-spread 0.2
```

Поосевой разброс (четыре σ):

```bash
uv run python run_genetic_optimization.py --initial-weights "1,0,0,0" --initial-spread-per-weight "0.3,0.2,0.2,0.1"
```
